# قوموا لله — موقع استلام تقديمات الصور

موقع + خادم (سيرفر) لاستقبال الأعمال الفوتوغرافية عبر استمارة، وإرسالها بشكل خاص إلى بوت تلكرام.
توكن البوت محفوظ على الخادم فقط ولا يظهر أبداً في كود الصفحة، على عكس نسخة الملف الواحد السابقة.

## 1. إنشاء بوت تلكرام (إذا لم يكن جاهزاً)

1. افتح تلكرام وابحث عن **@BotFather**
2. أرسل `/newbot` واتبع التعليمات (اسم البوت، ثم اسم مستخدم ينتهي بـ `bot`)
3. احفظ **التوكن** الذي يعطيك إياه (شكله: `123456789:ABCdefGhIJKlmNoPQRstuVwxyZ`)
4. ابحث عن **@userinfobot** وابدأ محادثة معه ليعطيك **Chat ID** الخاص بك
5. أرسل أي رسالة إلى بوتك الجديد (حتى يستطيع مراسلتك لاحقاً)

## 2. التثبيت والتشغيل محلياً

يتطلب [Node.js](https://nodejs.org) نسخة 18 أو أحدث.

```bash
# داخل مجلد المشروع
npm install

# انسخ ملف الإعدادات وعدّل القيم
cp .env.example .env
```

افتح ملف `.env` وضع فيه:

```
BOT_TOKEN=التوكن_الخاص_بك
CHAT_ID=معرف_المحادثة_الخاص_بك
PORT=3000
```

ثم شغّل الخادم:

```bash
npm start
```

افتح المتصفح على: `http://localhost:3000`

## 3. النشر على الإنترنت (استضافة)

المشروع عبارة عن تطبيق Node.js/Express عادي، ويعمل على أي استضافة تدعم Node، مثل:

- **Render.com** (يوجد خطة مجانية) — اربط المستودع، واختر "Web Service"، وضع متغيرات البيئة `BOT_TOKEN` و `CHAT_ID` من لوحة التحكم
- **Railway.app**
- أي VPS (مثل خادم أوبنتو) عبر تشغيل `npm start` خلف Nginx أو باستخدام PM2 لإبقائه يعمل باستمرار

**مهم:** في كل هذه الخيارات، لا تضع ملف `.env` نفسه على الاستضافة — بل أضف `BOT_TOKEN` و `CHAT_ID` كمتغيرات بيئة (Environment Variables) من لوحة تحكم الاستضافة مباشرة، فهذا أكثر أماناً.

### النشر المجاني على Render (خطوة بخطوة)

1. ارفع محتوى هذا المجلد إلى مستودع (Repository) على GitHub
2. أنشئ حساباً مجانياً على [render.com](https://render.com) وسجّل دخول عبر GitHub
3. من لوحة التحكم اختر **New → Web Service** واختر المستودع الذي رفعته
4. Render سيكتشف تلقائياً أنه مشروع Node.js (بفضل ملف `render.yaml` المرفق) ويقترح:
   - Build Command: `npm install`
   - Start Command: `npm start`
5. اختر الخطة **Free**
6. في قسم **Environment Variables** أضف:
   - `BOT_TOKEN` = التوكن الخاص بك
   - `CHAT_ID` = معرف المحادثة الخاص بك
7. اضغط **Create Web Service** وانتظر انتهاء البناء (دقيقة أو دقيقتين)
8. سيعطيك Render رابطاً جاهزاً مثل: `https://qumu-lillah-submissions.onrender.com`

**ملاحظة عن الخطة المجانية:** الخدمة تدخل في وضع سكون بعد 15 دقيقة من عدم الاستخدام، وأول طلب بعدها يأخذ حوالي 30-60 ثانية حتى يعمل الموقع من جديد. هذا طبيعي في الخطط المجانية ولا يؤثر على البيانات أو الأمان.

## ملاحظات أمان

- التوكن ومعرّف المحادثة موجودان فقط في `.env` على الخادم، ولا يصلان أبداً إلى متصفح الزائر
- الخادم يتحقق من نوع وحجم كل صورة مرفوعة قبل قبولها (JPG/JPEG/PNG فقط، حتى 20 ميغابايت لكل صورة)
- الصور تُرسل عبر `sendDocument` / `sendMediaGroup` للحفاظ على جودتها الأصلية دون ضغط

## تقديم واحد لكل شخص + صور متعددة

- الاستمارة تسمح برفع **حتى 10 صور دفعة واحدة**، ببيانات (اسم، هاتف، إيميل...) **مرة واحدة فقط** لكل تقديم
- الخادم يمنع أي شخص من التقديم أكثر من مرة، عبر مطابقة **عنوان IP + البريد الإلكتروني + رقم الهاتف** (تكرار أي واحد منهم يكفي لرفض المحاولة)
- البيانات المستخدمة للمطابقة تُحفظ **مشفّرة (hashed)** فقط في `data/submissions.json`، وليست بيانات خام
- **قيد مهم:** التحقق عبر IP ليس مثالياً 100% — أشخاص خلف نفس الشبكة (مثل واي فاي عام أو شبكة شركة) قد يُعتبرون شخصاً واحداً، بينما شخص يستخدم VPN يقدر يغيّر IP الخاص به. لذلك التحقق بالبريد الإلكتروني ورقم الهاتف هو الأكثر موثوقية.
- **تنبيه استضافة:** إذا نشرت الموقع على استضافة مجانية بتخزين مؤقت (Ephemeral) مثل Render المجاني، فإن ملف `data/submissions.json` قد يُمسح عند إعادة تشغيل الخادم (بعد فترات خمول طويلة أو تحديثات). إذا كان منع التكرار أمراً حساساً جداً لمشروعك، أخبرني لأضيف قاعدة بيانات دائمة (مثل Render الـ PostgreSQL المجاني).
